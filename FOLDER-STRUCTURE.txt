Wireshark Teaching Site — intended folder structure
====================================================

This is the TARGET layout. You don't create these by hand — Claude Code will
generate them. This is just so you can eyeball the shape and tell Claude Code
if you want it organized differently.

wireshark-lab-site/
├── CLAUDE.md                 # project context (already created — read every session)
├── README.md                 # instructor lab worksheet (already created)
├── package.json              # deps + npm scripts (start:http, start:https, gen-cert)
├── .claude/
│   └── settings.json         # optional: permissions.allow list so the agent
│                             #   doesn't prompt on every npm/git command
├── certs/                    # self-signed cert for the HTTPS twin
│   ├── key.pem               #   (generated by `npm run gen-cert`, gitignored)
│   └── cert.pem
├── src/
│   ├── server-http.js        # plain HTTP server on :8080
│   ├── server-https.js       # HTTPS twin on :8443 (loads certs/)
│   ├── app.js                # shared Express app: all routes mounted here
│   ├── guard.js              # startup private-IP guard (--allow-public to override)
│   └── routes/
│       ├── login.js          # GET form + POST urlencoded creds, sets session cookie
│       ├── search.js         # GET query in URL
│       ├── register.js       # multi-field urlencoded form
│       ├── upload.js         # multipart/form-data (multer)
│       ├── api.js            # /api/data JSON response
│       ├── secret.js         # HTTP Basic Auth (Base64) protected page
│       └── status.js         # /broken (404), /redirect (301/302), /error (500)
├── views/                    # server-rendered pages (EJS or template strings)
│   ├── layout.ejs            # includes the persistent INSECURE banner
│   ├── index.ejs             # landing page + lesson index
│   ├── login.ejs
│   ├── register.ejs
│   ├── upload.ejs
│   ├── secret.ejs
│   └── assets-demo.ejs       # page that pulls several local assets
└── public/                   # LOCAL assets only (no CDNs) — keeps captures clean
    ├── css/style.css
    ├── js/main.js
    └── img/sample.png

Notes
-----
- Everything under public/ is served locally and deliberately small, so the
  only packets students capture are the lesson's own requests.
- certs/ should be gitignored; the cert is generated on each machine.
- The HTTP and HTTPS servers share the SAME app.js so the routes are identical
  — that's what makes the HTTP-vs-HTTPS capture contrast valid.
- Keep one route = one lesson. Resist merging them; the separation is what
  makes each Wireshark filter land cleanly.
